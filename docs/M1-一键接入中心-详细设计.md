# M1 · 一键接入中心（Integration Hub）详细设计

> **目标**：3 分钟把任意平台的 Agent / MCP 工具 / Skill 接入平台，并自动生成一名"数字员工"。
>
> 这是整个平台的入口模块，承担"协议中立适配器"的角色。

---

## 1. 设计目标

| 目标 | 衡量标准 |
| --- | --- |
| 极速接入 | 从"选平台"到"数字员工上架" ≤ 3 分钟，≤ 5 步 |
| 协议中立 | OpenClaw / Hermes / Cursor / Claude / 自定义 一套统一流程 |
| 能力无损 | 接入侧所有工具 / Skill 都能被发现并绑定到员工 |
| 可断点续接 | 任意步骤可中断、可恢复，不丢上下文 |
| 自动生成 | 接入完成自动产出 Agent Card + 数字员工简历 |

---

## 2. 接入流程状态机

接入一名数字员工的生命周期共 **6 个状态**：

```
                  ┌─────────────┐
   选平台 ───────▶ │  pending    │  用户已选定平台连接器，未开始
                  └──────┬──────┘
                         │ startAuth()
                         ▼
                  ┌─────────────┐    失败/超时
   鉴权中  ◀───── │ authenticating │ ────────▶  failed  ──▶ pending (重试)
                  └──────┬──────┘
                         │ auth callback ok
                         ▼
                  ┌─────────────┐
                  │ discovering │  调 tools/list、skills/sync 发现能力
                  └──────┬──────┘
                         │ discover ok
                         ▼
                  ┌─────────────┐
                  │  composing  │  用户勾选能力 + 填人设 + 定价
                  └──────┬──────┘
                         │ publish()
                         ▼
                  ┌─────────────┐
                  │  connected  │  数字员工已生成、已上架，可被雇佣
                  └──────┬──────┘
                         │ unbind() / error
                         ▼
                  ┌─────────────┐
                  │ disconnected│  解绑，员工下架但记录保留
                  └─────────────┘
```

### 状态定义

| 状态 | 含义 | 可流转到 | 持久化要点 |
| --- | --- | --- | --- |
| `pending` | 已选平台，未鉴权 | `authenticating` | 保存 `platform`、`connector_id` |
| `authenticating` | Device Auth 进行中 | `discovering` / `failed` | 保存 `device_code`、`expires_at` |
| `discovering` | 发现能力中 | `composing` / `failed` | 调外部协议，不持久化 |
| `composing` | 用户组装数字员工 | `connected` / `pending` | 草稿态 `draft_employee` |
| `connected` | 正常工作 | `disconnected` | 数字员工已上架 |
| `disconnected` | 已解绑 | `pending`（重新接入） | 员工下架，记录归档 |
| `failed` | 鉴权 / 发现失败 | `pending`（重试） | 保存 `error_code`、`error_msg` |

### 状态迁移规则

- 任意状态都可在 `failed` / `disconnected` 后回退到 `pending` 重新开始。
- `composing` 是**唯一允许长时间停留**的状态（用户填表）—— 用 `draft_id` 持久化草稿，支持断点续填。
- `connected` 是终态之一，只有在用户主动 `unbind` 或鉴权令牌过期（不可刷新）时才进入 `disconnected`。

## 3. 各平台连接器接口契约

所有连接器实现同一接口 `IConnector`，平台差异封装在内部。

### 3.1 统一连接器接口

```ts
interface IConnector {
  // 平台标识，对齐 platform_bindings
  platform: 'openclaw' | 'hermes' | 'cursor' | 'claude' | 'custom';

  // 1. 发起 Device Auth（返回 device_code + 用户可访问的验证入口）
  startAuth(input: StartAuthInput): Promise<StartAuthOutput>;

  // 2. 轮询鉴权结果（或回调）
  pollAuth(device_code: string): Promise<AuthResult>;

  // 3. 发现能力（MCP tools/list、Skill 列表、A2A 端点）
  discoverCapabilities(access_token: string): Promise<DiscoveredCapability[]>;

  // 4. 健康检查 / 令牌续期
  ping(binding: PlatformBinding): Promise<{ ok: boolean; expires_at?: number }>;

  // 5. 解绑（撤销授权）
  unbind(binding: PlatformBinding): Promise<void>;
}
```

### 3.2 通用 DTO

```ts
interface StartAuthInput {
  redirect_uri?: string;       // 回调地址（Web 流程）
  scope?: string[];            // 申请的能力范围
}

interface StartAuthOutput {
  device_code: string;         // 平台返回的设备码
  user_code?: string;          // 展示给用户的短码（扫码流）
  verification_url?: string;   // 用户需访问的验证页
  qr_url?: string;             // 二维码图片 URL
  expires_in: number;          // 秒
  interval: number;            // 轮询间隔（秒）
}

interface AuthResult {
  status: 'pending' | 'success' | 'expired' | 'denied';
  access_token?: string;
  refresh_token?: string;
  platform_user_id: string;    // 该平台侧的用户/Agent ID（即 X-Platform-User-Id）
  expires_at: number;          // unix 秒
}

interface DiscoveredCapability {
  kind: 'mcp_tool' | 'skill' | 'a2a_endpoint' | 'skill_pack';
  id: string;                  // 平台侧唯一 ID
  name: string;
  description: string;
  schema?: object;             // 入参 JSON Schema
  category?: string;           // 分类，用于挂标签
}
```

### 3.3 各平台差异速查

| 平台 | Auth 方式 | 能力发现 | 备注 |
| --- | --- | --- | --- |
| **OpenClaw**（推荐） | Device Auth（扫码） | `skills/list` + MCP `tools/list` | 默认 scope=`skill_pack:read tools:read` |
| **Hermes** | Device Auth | MCP `tools/list` | 平台类型固定为 `hermes_agent`，配置项需带 `platform_type` |
| **Cursor** | 本地 MCP 配置注入（无 OAuth） | 解析 `~/.cursor/mcp.json` 后 `tools/list` | 需用户手动粘贴配置或上传文件 |
| **Claude Desktop** | 同 Cursor，配置在 `claude_desktop_config.json` | `tools/list` | 桌面端本地接入，无远程令牌 |
| **Custom** | 用户自填 `base_url` + `api_key` | 优先 `/agent_card`，退化到 `tools/list` | 兜底，对接任意 MCP/A2A 服务 |

## 4. Device Auth 绑定流程

采用 OAuth 2.0 **Device Authorization Grant**（RFC 8628）变体，适合 Agent 这类无浏览器的客户端。

```
┌──────────┐   1. POST /integrations/auth/start        ┌──────────┐
│  Client  │ ─────────────────────────────────────────▶│WorkTwin API│
│ (用户端) │ ◀────────── device_code + user_code ──────│          │
└──────────┘                                            └────┬─────┘
      │                                                      │
      │  2. 用户打开 verification_url，扫码/输码完成授权       │
      │                                                      │
      │   3. 客户端按 interval 轮询                            │
      │   POST /integrations/auth/poll {device_code}          │
      │ ─────────────────────────────────────────────────▶    │
      │ ◀──── status: pending / success(access_token) ────────│
```

- 成功后 `access_token` 与 `platform_user_id` 落库为一条 `PlatformBinding`。
- `access_token` 过期前 5 分钟由后台 `ping()` 触发刷新；刷新失败 → 状态回退 `failed`，通知用户重新授权。

## 5. Skill Pack 导入与 MCP 工具同步

### 5.1 Skill Pack 导入

Skill Pack = 一组打包好的技能（如"数据分析套装"=清洗 + 可视化 + 报告）。

```http
POST /integrations/{binding_id}/skill-packs/import
Content-Type: application/json

{
  "pack_id": "sp_data_analysis_v2",
  "auto_bind": true               // 是否自动全部绑定到草稿员工
}
```

响应里返回导入成功的 `skill_id[]`，失败项单独列出（部分成功允许）。

### 5.2 MCP 工具同步

后台定时（默认每 10 分钟）对每个 `connected` 状态的绑定调一次 `tools/list`，diff 后：

- **新增工具** → 进 `DiscoveredCapability` 队列，提示用户"是否加入数字员工"。
- **移除工具** → 标记 `deprecated`，已绑定员工的对应能力进入"待替换"状态，**不立即中断在岗任务**。
- **Schema 变更** → 版本号 +1，影响范围内的在岗任务打 `compatibility_warning`。

### 5.3 A2A 端点注册

```http
POST /integrations/{binding_id}/a2a-endpoints
{ "url": "https://my-agent.com/a2a", "methods": ["tasks/send","tasks/get"] }
```

注册后该端点会写进数字员工的 `agent_card.endpoints`，供其他 Agent 通过 A2A JSON-RPC 调用。

## 6. REST API 端点设计

所有响应遵循开发者文档统一信封：

```json
{ "code": 0, "message": "success", "data": T, "timestamp": 1700000000 }
```

### 6.1 接入会话

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `POST` | `/integrations/sessions` | 创建接入会话（选平台）→ `{session_id, status: 'pending'}` |
| `GET` | `/integrations/sessions/{id}` | 查询会话状态（含当前 state） |
| `DELETE` | `/integrations/sessions/{id}` | 放弃接入，回退 `pending` 或归档 |

### 6.2 鉴权

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `POST` | `/integrations/sessions/{id}/auth/start` | 发起 Device Auth → 返回 `device_code` 等 |
| `POST` | `/integrations/sessions/{id}/auth/poll` | 轮询鉴权结果 |
| `POST` | `/integrations/sessions/{id}/auth/callback` | OAuth 重定向回调（Web 流程） |

### 6.3 能力发现与选择

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/integrations/sessions/{id}/capabilities` | 列出 `discovering` 抓到的能力 |
| `POST` | `/integrations/sessions/{id}/capabilities/select` | 勾选要绑定的能力，进入 `composing` |

### 6.4 数字员工草稿与发布

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `PUT` | `/integrations/sessions/{id}/draft` | 写入 / 更新人设、定价等草稿字段 |
| `POST` | `/integrations/sessions/{id}/publish` | 发布 → 生成数字员工，状态转 `connected` |

### 6.5 绑定管理

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/integrations/bindings` | 列出当前用户所有平台绑定 |
| `POST` | `/integrations/bindings/{id}/refresh` | 手动刷新令牌 / 重新同步工具 |
| `DELETE` | `/integrations/bindings/{id}` | 解绑 → 数字员工下架，状态 `disconnected` |

## 7. 错误码与边界场景

### 7.1 错误码

| code | 含义 | 触发场景 | 用户可读提示 |
| --- | --- | --- | --- |
| `40001` | 平台不支持 | `platform` 不在枚举内 | "暂不支持该平台，请选择 OpenClaw / Hermes / Cursor / Claude / Custom" |
| `40002` | Device Auth 超时 | `expires_in` 内未完成授权 | "授权超时，请重新发起" |
| `40003` | 鉴权被拒 | 用户在平台侧拒绝授权 | "授权被拒绝，可重新发起" |
| `40004` | 能力发现失败 | `tools/list` 等调用失败 | "无法读取该平台的能力列表，请检查权限" |
| `40005` | 令牌失效 | `access_token` 过期且无法刷新 | "登录已失效，请重新授权" |
| `40006` | 草稿不完整 | `publish` 时必填字段缺失 | "请补全姓名 / 岗位 / 定价后再发布" |
| `40007` | 平台侧限流 | 上游 429 | "平台繁忙，稍后自动重试" |
| `40008` | 部分能力导入失败 | Skill Pack 部分项失败 | "N 项技能导入失败，已跳过：…" |

### 7.2 边界场景

- **令牌过期且在岗任务进行中**：不立即中断任务；任务结束后再标记 `disconnected`，通知用户续授权。
- **同一平台重复接入**：允许（例如同一用户接两个 OpenClaw 账号），生成不同 `PlatformBinding`，但数字员工默认去重（同名同能力提示合并）。
- **发现到 0 个能力**：允许进入 `composing`，但 `publish` 时校验至少 1 个能力，否则报 `40006`。
- **用户中途换平台**：从 `composing` 之前的状态可直接 `DELETE session` 重建；`composing` 之后需先"丢弃草稿"才能换。
- **网络抖动**：`poll` 接口要求客户端遵守返回的 `interval`，过快轮询返回 `40007`。

## 8. 与其他模块的衔接

| 衔接模块 | 衔接点 | 说明 |
| --- | --- | --- |
| **M2 数字员工工坊** | `publish()` 产出 | 接入中心生成的员工进入工坊做精修 |

## 9. P1 MVP 简化范围

为快速跑通主链路，P1 阶段做以下裁剪：

| 项目 | P1 实现 | P2 补齐 |
| --- | --- | --- |
| 平台连接器 | 仅 OpenClaw + Custom（共 2 个） | Hermes / Cursor / Claude 全量 |
| Auth 方式 | Custom 用"粘贴 api_key"；OpenClaw 用模拟扫码 | 真 Device Auth + 令牌刷新 |
| 能力发现 | 从 `lib/store.ts` 种子数据返回 | 真调 `tools/list` / `skills/list` |
| Skill Pack 导入 | 单技能逐个导入 | 批量 Pack 导入 |
| 同步 | 手动刷新按钮 | 定时后台同步 + diff |
| 断点续接 | 草稿存 localStorage | 草稿落库，跨设备恢复 |

P1 的目标只有一个：**让"选平台 → 鉴权 → 选能力 → 生成员工"这条链路在前端能点完、点通**。
