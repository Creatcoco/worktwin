# WorkTwin 商业化上线清单

> 更新日期：2026-06-20

## 当前可用范围

当前仓库已经达到“可演示、可验证产品闭环”的前端 MVP 水平：

- 首页主张、市场、员工详情、工坊、工作台、派单、结算、开发者中心可完整浏览。
- OpenClaw、Hermes、Cursor、Claude、Custom 均有独立接入参数、预检和能力发现流程。
- 接入发布会真实生成数字员工，并进入市场、工坊和 Agent Card。
- 雇佣会生成合同；合同支持暂停/恢复。
- 派单会校验合同和钱包余额；任务完成后自动生成结算并扣减钱包。
- CNY 与 UT 分账统计，不再混合口径。
- API Key 默认遮罩并支持安全随机轮换。
- `npm run lint` 与 `npm run build` 均通过。

## P0：对外收费前必须完成

### 身份与权限

- 接入真实登录：邮箱/手机号/OAuth，支持 MFA。
- 服务端实现 session、JWT、API Key hash、密钥轮换和撤销。
- 所有资源按 tenant/user 做行级权限校验，禁止依赖前端 ownerId。
- 管理员、雇主、员工所有者、Agent 调用方使用独立 RBAC 权限。

### 数据持久化

- 将 `web/lib/store.ts` 替换为 PostgreSQL + ORM repository。
- 为合同、任务、结算、连接会话增加数据库事务和唯一约束。
- 为状态变更增加 append-only audit log。
- 为任务、回调、结算建立幂等键，避免重复扣款和重复交付。

### 支付与资金

- CNY 接入合规支付渠道，不在平台内直接模拟真实余额。
- UT 明确法律属性、购买/消耗/退款规则，避免被误解为可交易代币。
- 使用预授权/托管：派单冻结预算，验收后结算，争议期内可退款。
- 财务流水与业务订单分表，并提供日终对账、差错处理和发票能力。

### Agent 运行安全

- 连接器 token 仅在服务端加密保存，前端不得持久化明文。
- `exec`、文件写入、浏览器提交、发消息、上传等能力默认人工确认。
- 每个任务使用隔离 worker/sandbox，设置 CPU、内存、网络和执行时限。
- 对 prompt injection、数据外传、恶意 MCP server 做输入输出审查。

### 可靠性

- 连接器 probe、任务执行、webhook、结算全部进入持久队列。
- 定义重试、退避、死信队列、超时、熔断和补偿事务。
- 建立 SLO：API 可用性、任务成功率、P95 响应、结算正确率。
- 接入错误监控、日志聚合、链路追踪和告警值班。

## P1：公测期补齐

- 免费面试/试用额度和反滥用策略。
- 合同有效期、用量上限、续约、解雇和退款。
- 任务验收、驳回、修改、评价和争议仲裁。
- 员工信用分、能力验证、身份认证、违规下架。
- 收益看板、平台抽成、提现、税务/发票处理。
- 团队与企业租户、成员权限、预算中心和审批流。
- 隐私政策、服务条款、数据处理协议和未成年人政策。

## 推荐后端边界

```text
Web / BFF
  -> Identity & Tenant Service
  -> Marketplace Service
  -> Contract & Task Service
  -> Connector Gateway
  -> Worker / Sandbox Fleet
  -> Billing & Ledger Service
  -> Trust & Safety Service
  -> Notification / Webhook Service
```

前端页面可以继续沿用当前模型，但所有写操作应改为调用服务端 API，客户端 store 只保留查询缓存和乐观更新。

## 上线判定

只有当以下条件全部满足，才可标记为“可商业化生产使用”：

- 没有明文生产密钥落到浏览器或日志。
- 重复请求不会重复扣费、重复结算或重复执行任务。
- 任意用户无法读取或修改其他租户的员工、合同、任务和账单。
- 高风险 Agent 工具始终经过策略判断、沙箱或人工确认。
- 支付、退款、提现和对账均有可审计流水。
- 关键流程有自动化测试、监控、告警和灾难恢复演练。
