# 飞书登录注册配置

WorkTwin 的注册、登录和当前用户校验通过服务端 Route Handler 访问飞书多维表格。浏览器不会接触 App Secret、tenant token、密码哈希或密码盐。

## 1. 创建飞书应用

1. 在 [飞书开放平台](https://open.feishu.cn/app) 创建企业自建应用。
2. 开通多维表格读写权限 `bitable:app`。
3. 发布应用版本。
4. 在目标多维表格的分享设置中，将应用添加为可编辑协作者。

飞书权限需要同时满足“应用权限范围”和“资源协作者授权”。

## 2. 创建用户数据表

创建一个多维表格和数据表，并按下表创建字段。字段名称必须完全一致。

| 字段 | 推荐类型 | 说明 |
| --- | --- | --- |
| 用户ID | 单行文本（主字段） | `usr_...` 唯一用户标识 |
| 邮箱 | 单行文本 | 统一保存为小写，登录查询键 |
| 姓名 | 单行文本 | 用户显示名称 |
| 密码哈希 | 单行文本 | scrypt 结果，不是明文密码 |
| 密码盐 | 单行文本 | 每个用户独立随机盐 |
| 状态 | 单选 | 选项：`active`、`disabled` |
| 角色 | 单选 | 选项：`user`、`admin` |
| 创建时间 | 日期 | 毫秒时间戳 |
| 更新时间 | 日期 | 毫秒时间戳 |
| 最后登录时间 | 日期 | 毫秒时间戳，可为空 |

从多维表格 URL 中获取 `app_token`，再通过飞书 API 或开发者工具获取形如 `tbl...` 的 `table_id`。

## 3. 配置凭证

复制示例文件：

```bash
cd web
cp config/feishu-auth.example.json config/feishu-auth.json
```

填写配置：

```json
{
  "app_id": "cli_xxxxxxxxxxxxxxxx",
  "app_secret": "replace_me",
  "tenant_access_token": "",
  "app_token": "bascnxxxxxxxxxxxxxxx",
  "table_id": "tblxxxxxxxxxxxxxxx",
  "session_secret": "replace_with_at_least_32_random_characters"
}
```

推荐填写 `app_id + app_secret`，服务端会自动获取并缓存两小时有效的 `tenant_access_token`。也支持直接填写 `tenant_access_token`，但过期后需要人工更新。

`session_secret` 可使用以下命令生成：

```bash
openssl rand -hex 32
```

真实配置文件 `web/config/feishu-auth.json` 已加入 `.gitignore`，禁止提交到仓库。

如需存储完整产品数据，还需在同一配置文件中增加 `tables` 子表映射，详见 [`飞书产品数据模型.md`](./飞书产品数据模型.md)。

生产部署也可使用环境变量：

```text
FEISHU_APP_ID
FEISHU_APP_SECRET
FEISHU_TENANT_ACCESS_TOKEN
FEISHU_AUTH_APP_TOKEN
FEISHU_AUTH_TABLE_ID
FEISHU_TABLES_JSON
AUTH_SESSION_SECRET
```

## 4. 验证配置

启动应用后访问：

```text
GET /api/auth/health
```

成功响应：

```json
{
  "ok": true,
  "requiredFields": ["用户ID", "邮箱", "姓名", "密码哈希", "密码盐", "状态", "角色", "创建时间", "更新时间", "最后登录时间"]
}
```

如果表格无权限、token 失效或字段缺失，接口会返回 503 和具体原因。

## 5. 已实现接口

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | `/api/auth/register` | 邮箱查重、scrypt 哈希、写入飞书、创建会话 |
| POST | `/api/auth/login` | 读取飞书用户、校验密码、更新最后登录时间 |
| POST | `/api/auth/logout` | 清除 HttpOnly 会话 Cookie |
| GET | `/api/auth/me` | 校验签名会话，并重新检查飞书用户状态 |
| GET | `/api/auth/health` | 检查凭证、表权限和字段结构 |

## 6. 安全说明

- 密码使用 scrypt + 每用户独立随机盐，只保存哈希和盐。
- 会话使用 HMAC-SHA256 签名，Cookie 为 HttpOnly、SameSite=Lax，生产环境启用 Secure。
- 登录和注册包含基础频率限制。
- 登录错误不区分“用户不存在”和“密码错误”，降低账号枚举风险。
- 用户被设置为 `disabled` 后，`/api/auth/me` 会拒绝其现有会话。

多维表格不提供数据库级邮箱唯一约束。正式高并发商用时，建议将认证主库迁移到 PostgreSQL/Auth 服务，飞书作为运营同步和管理后台。
